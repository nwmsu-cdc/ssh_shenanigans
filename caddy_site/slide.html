What is SSH

  • Usually on port 22
  • Client <-> Server

Getting Your Tools! :)

After downloading the tools from the lab server

cd Downloads
sh ssh_tools.sh

Accessing a Server



ssh username@server

On our server SSH is hosted on port 23!!!

ssh admin@server_ip -p 23

Config File

Having to type -p 23 is cringe. Passwords are also cRinGe!

. . .

Lets use a config instead

Host bastion
        HostName HOST_IP
        User admin
        Port 23

Identity Files :)

cd ~/.ssh
ssh-keygen -t ed25519 # After you can type the name of the file

. . .

Host bastion
        HostName HOST_IP
        User admin
        Port 23
        IdentityFile ~/.ssh/bastion

Running Remote Commands

ssh bastion "sleep 10; hostname" # runs and keeps you hostage
ssh bastion -f "sleep 10; hostname" # runs and lets you be FREE!!!!!

Dynamic Proxies :3

ssh -D [port] [host] -fN # fork with no command

Set up a socks proxy on a port ( you can configure firefox to use it too! )

. . .

ssh -D 9050 bastion -fN 

# Proxychains defaults to 9050 (/etc/proxchains.conf)
proxychains curl 10.5.0.5 # curls from the servers PoV

Jump Hosts

 ssh -J bastion admin@internal # connects to internal via bastion
Host bastion
        HostName HOST_IP
        User admin
        Port 23
        IdentityFile ~/.ssh/bastion

Host internal
        HostName 10.6.0.10
        User admin
        ProxyJump bastion
        IdentityFile ~/.ssh/internal
